## Successes

1. **Zero-debt-forward upheld for 4th consecutive retro** — 14/14 prior items delivered, 37 new findings fully assigned
2. **Three-vector computation model (8.0a)** — foundational domain-correct architecture; scheme formula (Px13.33%/60) used by every downstream service
3. **Auto-stop full cycle delivered (8.1→8.2→8.3)** — detect zero balance → generate certificate with QR → notify MDA + beneficiary. The core promise.
4. **Coverage Tracker praised by AG** — only positive UAT finding. MDA x Month grid with clickable cells + CSV/Excel download
5. **Selective baseline pipeline (8.0j)** — most complex prep story (10 ACs). Bridges authority/knowledge gap between DEPT_ADMIN and MDA_ADMIN
6. **Zero production incidents** — down from 2 in E7+E6. "Deploy = verify" agreement working
7. **Code review findings down 23%** (91 vs 118) — quality improving at the source
8. **Period confirmation gate (8.0d)** — PM elevated passive badge to active gate. Prevents wrong-month uploads

---

## Challenges

1. **MDA officer experience fundamentally under-served** — sparse dashboard (5 sidebar items), hidden workflows, no upload capability, no migration quality visibility
2. **Dead-end metrics everywhere** — 7+ findings where users see numbers but can't click into them
3. **apiClient unwrap bug (#5)** — critical, affects all roles, loans table completely broken. Invisible to 2,132 tests
4. **Observations not auto-triggered after baseline (#8)** — observation engine exists but nobody wired it to the baseline completion event
5. **AG-centric upload model doesn't scale** — current design has AG uploading for 47 MDAs. Federated model needed
6. **Server-heavy build** (3.2:1 ratio) — backend infrastructure solid but frontend UX debt accumulated
7. **Public certificate verification page missing (#33)** — route defined, component never created. QR codes lead to broken page

---

## Key Insights

1. **UAT catches what 2,132 tests and 91 code review findings don't** — 37 UAT findings including a critical bug invisible to automated testing
2. **The data pipeline must be federated** — MDA officers as first-class data contributors, not passive reviewers. Infrastructure 90% built
3. **Every visible number must be a doorway** — the AG's mental model: "I see it, I click it, I see the names." No dead-end metrics
4. **Role-specific testing is essential** — AG, Dept Admin, and MDA Officer see completely different realities. Testing one role misses the others
5. **Domain nuance drives architecture** — consecutive loans, transfers, within-file duplicates all need explicit handling. The data model supports it but the UX doesn't surface it
6. **Prep stories are scope DISCOVERY, not scope creep** — every addition validated by UAT evidence, not speculation
