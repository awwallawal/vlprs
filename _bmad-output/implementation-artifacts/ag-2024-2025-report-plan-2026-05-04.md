## 4. The 20-column schema — first-class vs. computed vs. inferred

| Column | Source | Risk |
|---|---|---|
| S/No | row index | trivial |
| Name | `record.name` | safe |
| Commencement Date | **inferred** — `record.startDate` if present, else back-derive | mostly null in catalog |
| Completion Date | **inferred** — Commencement + installmentCount months | depends on Commencement |
| LPC Out | **synthesised** per Q3 | not a PRD concept, judgement-dependent |
| No of Installment | `record.installmentCount` (chosen tenure) | safe — confirmed Q4-class with Dana |
| Principal | `record.principal` | safe |
| Total Interest | `record.interest` | safe |
| Total Loan | `record.totalLoan` (also: Principal + Total Interest, must reconcile) | enforce internal consistency |
| Monthly Deduction | `record.monthlyDeduction` | safe |
| Monthly Interest | computed: (Principal × 0.1333) ÷ 60 | always ÷ 60 per scheme model |
| Monthly Principal | computed: Principal ÷ installmentCount | safe |
| No of Installment Paid | `record.installmentsPaid` | NULL on 96% of SPORTS COUNCIL pre-fix; check after Phase B |
| Total Interest Paid | computed: Monthly Interest × installmentsPaid | depends on installmentsPaid |
| Total Principal Paid | computed: Monthly Principal × installmentsPaid | depends on installmentsPaid |
| Total Loan Paid | computed: Monthly Deduction × installmentsPaid | depends on installmentsPaid |
| No of Installment Outstanding | `record.installmentsRemaining` | depends on extractor |
| Total Outstanding Interest | computed: Monthly Interest × installmentsRemaining | depends |
| Outstanding Principal Loan | computed: Monthly Principal × installmentsRemaining | depends |
| Total Outstanding Balance | `record.outstandingBalance` (also: Monthly Deduction × installmentsRemaining, reconcile) | NULL on 100% of SPORTS COUNCIL pre-fix |

**Internal-consistency rule:** every report row must satisfy:
- `Total Loan == Principal + Total Interest` (±₦1)
- `Monthly Deduction == Monthly Interest + Monthly Principal` (±₦1)
- `Total Loan Paid == Total Interest Paid + Total Principal Paid` (±₦1)
- `Total Outstanding Balance == Total Outstanding Interest + Outstanding Principal Loan` (±₦1)
- `installmentCount == installmentsPaid + installmentsRemaining` (exact)

Rows that violate any of these get flagged in a `_consistency-violations` sheet, not silently dropped.

## 15. Known unknowns / risks to flag explicitly

- **Multi-month files** (e.g., `APRIL TO JUNE,2025.xlsx`): if the file's data rows contain a period column, the extractor can split. If not, all rows go under the *first* month named — overstating that month, undercounting the others. Phase B.2 must surface this clearly to Awwal.
- **`Oyo_Housing_Corporation_2023-2025 VEHICLE LOAN DEDUCTION.xlsx` is 15.2 MB** — likely tens of thousands of rows spanning 3 years. May contain non-deduction sections (HR data, summary pages). Pre-ingest inspection mandatory before B.3 alias is added.
- **Mineral Development Agency file is named `*_CAR LOAN TRAINING.xlsx`** — "TRAINING" is suspicious. Could be a training manual, not deduction data. Inspect before treating it as a regular MDA submission.
- **`installmentsPaid` is null on most pre-2023 MDA records** beyond just SPORTS COUNCIL. The Total *Paid columns will frequently be blank, even after Phase B fixes. Cover letter must say so.
- **Cross-MDA name matching** uses normalised name only at this stage. Two distinct people with the same name could be conflated as one transfer. Phase E.3 should surface ambiguous cases in `_pending-transfer-cases` rather than commit to a transfer interpretation.
- **The 3 classification discrepancies (ESTABLISHMENT, GOVERNOR OFFICE, WOMEN AFFAIRS)** must be re-checked post-Phase-B. WOMEN AFFAIRS especially — if it's still "active classification, zero data" after a clean rerun, that's a separate bug in `car-loan-mda-classification.ts`.
