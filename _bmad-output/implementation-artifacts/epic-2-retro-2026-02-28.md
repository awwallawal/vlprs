### 3. Domain Knowledge Confirmed

- **13.33% is the universal standard rate** — non-negotiable. All 577 non-standard rate loans are genuine variances. No rate table needed — single threshold
- **No known netting policy** — OLANIYAN case raised as open question for AG clarification
- **CDU is autonomous** — independent department under Agriculture. Dual submission was reporting mix-up. Model as independent MDA with parent relationship
- **Missing MDAs** — assume unreadable/missing CDs. Track as "Data Pending" (neutral). Awwal continues archive recovery
- **Stalled deductions** — most likely cause is transfer gap (outgoing MDA signed off, incoming MDA hasn't acknowledged)

### 4. Bidirectional Transfer Lifecycle — Pulled from Phase 3 to MVP

Three-path transfer resolution:

| Path | Initiator | Counterpart | Fallback |
|------|-----------|------------|----------|
| Transfer Out | Outgoing MDA | Incoming MDA confirms | Dept Admin override |
| Claim Transfer In | Incoming MDA | Outgoing MDA confirms | Dept Admin override |
| Direct reassignment | Department Admin | Neither MDA involved | Migration + unresponsive MDAs |

Distribution across sprints:
- **Epic 3:** Department Admin direct reassignment (migration use case)
- **Epic 11 (Story 11.2):** FR61 extension — bidirectional transfer events (Transfer Out + Claim Transfer In), Transfer Search with scoped cross-MDA visibility
- **Epic 9:** Transfer notifications + escalation after X days unconfirmed

### 5. New Observation Type — Consecutive Loan Without Clearance

The OLANIYAN case proved the "gaming the system" pattern discussed during PRD brainstorming is real: 3 loan cycles, new loan disbursed while prior loan had ₦80,993 outstanding. Added as 6th observation type with non-punitive framing: *"Consecutive Loan Cycle — New loan ({amount}) commenced while prior loan had {balance} outstanding. This pattern is noted for committee review."*
