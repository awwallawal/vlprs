## Successes

1. **Non-punitive vocabulary discipline** held across all 4 epics. Every code review checked for it. Product identity decision that stuck.
2. **SQ-1 port strategy** — battle-tested algorithms ported from legacy scripts. Regression fixture suite (3.0a) provided automated correctness baseline across all Epic 3 stories.
3. **Test suite grew 123%** (531 → 1,186 server-side). Zero regressions. Code review process is the quality gate.
4. **Forward-investment pattern** — mdaAggregationService, loanClassificationService, revenueProjectionService built in 4.1, consumed unchanged across 4.2-4.4.
5. **Pipeline composability** — processSubmissionRows() extraction (5.2) enabled CSV, manual entry, comparison, and historical upload to share the same pipeline.
6. **transitionLoan refactored** to accept optional existingTx (11.2) — solved deadlock, made loan state machine composable.
7. **UAT drove scope improvement** — Awwal's Department Admin walkthrough revealed navigation gap, cascaded into 11.0a remediation + 11.0b coverage tracker + scope growth.
8. **Finding rate dropped 39%** — from 12.9/story (E3) to 7.9/story (E11). Dev agent learning from review feedback.

## Challenges

1. **Hollow/tautological tests** — systemic #1 risk. 14 of 27 stories had tests marked complete but missing, tautological, or testing mocks not production code. NOT improving (60% → 50% → 33% → 57%).
2. **Transaction scope errors** — 5 stories with tx bugs. Dev agent doesn't consistently reason about transaction boundaries. Most severe: 11.2 deadlock.
3. **N+1 query patterns** — 8 stories affected. Worst: 500+ queries for 63 MDAs (4.3). Consistently caught and fixed in review.
4. **Type/schema drift between layers** — DB columns, TS types, service expectations, frontend consumption disagreeing. 10+ stories.
5. **File list documentation inaccuracy** — 12 of 27 stories. Zero improvement from E10 team agreement.
6. **Column mapping gaps** — `OUTSTANDING BAL.` abbreviation not matched (surfaced in UAT). Summary sheets processed as data (double-counting).
7. **Missing grade-level vs loan-tier cross-validation** — engine validates by rate, not by principal vs staff grade.

## Key Insights

1. **UAT catches what 1,771 tests don't.** Tests verify code paths. UAT verifies user journeys through actual role-scoped accounts. Both are needed. The migration button incident (11.0a) proved this definitively.
2. **The code review process IS the quality gate** — 274 findings caught across 27 stories, including 26 CRITICALs. Goal is to keep catching, not to stop finding.
3. **Product scope is data-driven** — SQ-1 findings reshaped Epic 3 (5→10 stories), UAT findings added stories to E11. Real data confronting assumptions is where valuable scope discovery happens.
4. **Accelerated repayment pathways are real and common** — 42 of 107 BIR records at 11.11% (50-month), 6 at 8.89% (40-month). Engine recognizes the rates but labels them as "observations" instead of "accelerated pathways."
5. **Zero-debt-forward is a maturity decision** — 19 tech debt items accumulated across 4 epics because each felt "too small to story." Prep story approach prevents this.
6. **Previous story intelligence decays at epic boundaries** — within-epic learning works (finding rate drops). Between epics, context resets. Prep stories bridge this gap.

## Architectural Patterns to Codify

1. **Pipeline Composability** — one pipeline, multiple entry points, defensive source guards
2. **Computed-Not-Stored** — pure compute functions, no cache invalidation, always-fresh
3. **Forward-Investment Services** — build to full interface even if current story needs half
4. **Fire-and-Forget Email** — outside transactions, dev mode logging, no blocking
5. **transitionLoan with optional existingTx** — any transaction-opening function accepts outer tx
